Visualization

Click any node to expand the graph. All nodes directly connected to the selected node are added to the visualization.

Hover over a node (or press and hold on mobile) to display its identifier:

  • Public AS: ASN
  • Private AS: AS ID
  • Router: Router ID

Node shapes

  • Public AS: Yellow circle
  • Private AS: Blue circle
  • Router: Green square

Cypher (Neo4j)

The graph uses the Neo4j Cypher query language.

Node labels

  • Public_AS
  • Private_AS
  • Router

Relationships

  • LINK – Router borders a Public AS.
  • LOCATED_IN – Router belongs to a Public or Private AS.
  • PEERS_WITH – Two Public ASes share a border connection.
  • PRIVATE_OF – Private AS belongs to a Public AS.

Example queries

Shows a single Public AS node with ASN 174. 

MATCH (a:Public_AS {asn: 174})
RETURN a

Shows the graph of a Public AS, all connected Private AS nodes, and any Router nodes that belong to the AS. However, this AS does not contain any routers. 

MATCH (a:Public_AS {asn: 174})
OPTIONAL MATCH m=(a)<-[:PRIVATE_OF]-(p:Private_AS)
OPTIONAL MATCH n=(a)<-[:LOCATED_IN]-(r:Router)
RETURN a, m, n

Retrieves a specific Private AS node by its ID. 

MATCH (p:Private_AS {as_id: "65000_174"})
RETURN p

Private AS IDs are strings, so remember to include quotation marks.

Shows a Private AS and all routers located inside it. 

MATCH m=(p:Private_AS {as_id: "65000_174"})<-[:LOCATED_IN]-(r:Router)
RETURN m

Retrieves a specific router node by its router ID. 

MATCH (r:Router {router_id: "150.150.150.150_150"})
RETURN r

Shows a router, its owning node, and optionally any other public AS it connects to. 

MATCH m=(r:Router {router_id: "150.150.150.150_150"})-[:LOCATED_IN]->(owner)
OPTIONAL MATCH n=(r)-[:LINK]->(border:Public_AS)
RETURN m, n

Shows all routers that are bordering multiple Public ASes. 

MATCH m=(border:Public_AS)<-[:LINK]-(r:Router)-[:LOCATED_IN]->(owner)
RETURN r
